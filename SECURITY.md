# Security Policy

## Supported Versions

**Versions of the project are currently being supported with security updates.**

| Version | Supported |
| ------- | --------- |
| 1.0.x   | ✅        |

## Vulnerability Reporting

Tell them where to go, how often they can expect to get an update on a reported vulnerability, what to expect if the vulnerability is accepted or declined, etc.

You can report security vulnerabilities by sending an email to [security@example.com](mailto:security@example.com). You also can use the [GitHub Security Advisories](https://docs.github.com/en/code-security/security-advisories) feature to report vulnerabilities.

When reporting a vulnerability, please include the following information:

* A detailed description of the vulnerability, including steps to reproduce it.
* The version of the project that is affected.
